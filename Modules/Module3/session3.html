1. O que é o $PROFILE

O $PROFILE é uma variável automática que guarda o caminho para um script .ps1 que é carregado sempre que abres o PowerShell.

Esse script permite-te configurar o ambiente de trabalho: atalhos, funções, mensagens, etc.

1.1. Verificar, criar e editar o ficheiro

Verificar se o ficheiro existe:

Test-Path $PROFILE

Criar o ficheiro (caso não exista):

New-Item -Type File -Path $PROFILE -Force

Abrir para editar:

code $PROFILE

2. Adicionar aliases personalizados

Podes criar aliases úteis e gravá-los no perfil:

Set-Alias ll Get-ChildItem

Estes comandos tornam-se permanentes se forem colocados dentro do ficheiro $PROFILE.

2.1. Exemplos de aliases úteis

Alias Comando equivalente Descrição
ll Get-ChildItem Listar ficheiros e pastas
la Get-ChildItem -Force Listar tudo, incluindo ficheiros ocultos
np notepad Abrir o Notepad
cdp Set-Location “C:\Projetos” Ir diretamente para a pasta de projetos
cdh Set-Location $HOME Ir para a pasta do utilizador
gcm Get-Command Ver comandos disponíveis
gmo Get-Module Ver módulos carregados
gsv Get-Service Listar serviços
sps Start-Process Iniciar um processo ou abrir um ficheiro
ip ipconfig Ver o IP local (via terminal externo)
wo Write-Output Escrever texto no terminal
wi Write-Information Mensagem de informação controlável
date Get-Date Mostrar data e hora atuais
who whoami Ver nome de utilizador atual (via cmd)
rmd Remove-Item Apagar ficheiro ou pasta
mdn mkdir “NovaPasta” Criar rapidamente uma nova pasta
ps Get-Process Listar processos ativos
kill Stop-Process Terminar um processo
help Get-Help Ver ajuda de comandos

Podes experimentar estes comandos no terminal antes de os gravar no perfil.

2.2. Usar aliases dentro de scripts

Os aliases são ótimos no terminal, mas em scripts partilhados com outras pessoas podem causar problemas.

Exemplo:

wo "Olá!"     # Funciona se tiveres um alias wo para Write-Output

Mas se outro utilizador não tiver esse alias definido, o script vai dar erro.

Solução: definir o alias dentro do próprio script

Se quiseres usar um alias temporário num script:

Set-Alias wo Write-Output

Coloca esta linha no início do script. O alias vai existir apenas enquanto o script estiver a correr.

Boa prática: usa o nome completo do comando nos scripts sempre que possível. Aliases são ótimos para produtividade pessoal, mas tornam o código menos legível para outras pessoas.

3. Tornar funções permanentes

Funções definidas no terminal desaparecem quando fechas a sessão.
Para as tornar permanentes, coloca-as no $PROFILE.

Exemplos:

function Hoje {
    Get-Date -Format "dddd, dd MMMM yyyy"
}

function Saudacao {
    Write-Output "Olá, $env:USERNAME! Hoje é $(Get-Date -Format 'dddd, dd MMMM yyyy')"
}

function PProjetos {
    Set-Location "C:\Utilizadores\$env:USERNAME\Documentos\Projetos"
}

function Google {
    Start-Process "https://www.google.com"
}

function UltimosEventos {
    Get-EventLog -LogName System -Newest 10
}

function Rede {
    ipconfig | Out-Host
}

function Limpar-Logs {
    Remove-Item -Path "$env:TEMP\*.log" -Force -ErrorAction SilentlyContinue
}

function Tamanho-Pasta {
    param([string]$pasta)
    (Get-ChildItem -Path $pasta -Recurse | Measure-Object -Property Length -Sum).Sum / 1MB
}

3.1. Adicionar documentação às tuas funções com Get-Help

Podes escrever ajuda personalizada para qualquer função tal como nos comandos do PowerShell, usando um bloco de comentário estruturado.

Este bloco deve vir antes da função.

Exemplo:

<# .SYNOPSIS Converte temperatura de Celsius para Fahrenheit.

.DESCRIPTION Recebe um valor em graus Celsius e devolve o valor convertido em Fahrenheit.

.PARAMETER Celsius Valor numérico em Celsius.

.EXAMPLE Converter-CelsiusParaFahrenheit -Celsius 25

.NOTES Função criada para fins educativos. #> function Converter-CelsiusParaFahrenheit { param([double]$Celsius) return ($Celsius * 1.8 + 32) }

Depois de gravares esta função no $PROFILE, podes usar:

Get-Help Converter-CelsiusParaFahrenheit

Ou só ver exemplos:

Get-Help Converter-CelsiusParaFahrenheit -Examples

✅ Esta prática é útil para tornar o terminal mais inteligente e a tua função mais compreensível para ti ou para colegas.

4. Ações automáticas ao iniciar o terminal

Podes adicionar qualquer código ao perfil que será executado automaticamente, como:

  • Mostrar a data
  • Escrever uma mensagem de boas-vindas
  • Importar módulos
  • Criar diretórios temporários

Exemplo:

Write-Output "Bom trabalho, $env:USERNAME! Hoje é $(Get-Date -Format 'dddd, dd MMMM yyyy')"

4.1. Boas práticas e backup do perfil

  • Mantém o conteúdo simples e testado
  • Comenta o código para lembrar o que faz
  • Guarda uma cópia do $PROFILE noutro local (ex: OneDrive)
  • Não coloques comandos destrutivos ou demorados no perfil
  • Usa try/catch para proteger trechos sensíveis

Podes testar alterações ao perfil com:

. $PROFILE

Isto recarrega o perfil sem precisares de fechar e reabrir o terminal.

5. Personalização avançada

Além de atalhos e funções, podes tornar o PowerShell mais acolhedor, produtivo e até divertido. Aqui ficam sugestões que deves considerar colocar no teu $PROFILE.

5.1. Personalizar o prompt (linha de comando)

Redefinir a função prompt permite mudar a aparência da linha onde escreves os comandos.

function prompt {
    $host.UI.RawUI.ForegroundColor = "Red"
    $hora = Get-Date -Format "HH:mm"
    Write-Host "[$hora] $env:USERNAME em $(Get-Location)" -ForegroundColor Red -NoNewline
    return "`n> "
}

5.2. Mensagens de boas-vindas 

Write-Host "Olá $env:USERNAME! Bem-vindo ao PowerShell." -ForegroundColor Yellow
Write-Host "Hoje é $(Get-Date -Format 'dddd, dd MMMM yyyy')" -ForegroundColor Gray

Mensagem dinâmica:

$hora = (Get-Date).Hour
if ($hora -lt 12) {
    Write-Host "Bom dia!" -ForegroundColor Green
} elseif ($hora -lt 18) {
    Write-Host "Boa tarde!" -ForegroundColor Yellow
} else {
    Write-Host "Boa noite!" -ForegroundColor Magenta
}

5.3. Algo divertido

Podes criar banners com arte ASCII:

Write-Host "╔═════════════════════════════╗" -ForegroundColor Cyan
Write-Host "║   BEM-VINDO AO POWERSHELL   ║" -ForegroundColor Cyan
Write-Host "╚═════════════════════════════╝" -ForegroundColor Cyan

ou

$banner = @'
____             _

| |_) / _ \ / _ \| '_ \ 
|  __/ (_) | (_) | | | |

5.4. Melhorar o histórico com PSReadLine

Este módulo já vem incluído e pode ser configurado no $PROFILE:

Set-PSReadLineOption -EditMode Windows
Set-PSReadLineOption -HistorySearchCursorMovesToEnd
Set-PSReadLineOption -HistorySaveStyle SaveIncrementally
Set-PSReadLineOption -HistorySavePath "$env:APPDATA\PowerShell\PSReadLine\History.txt"

O que faz cada linha?

  • Set-PSReadLineOption -EditMode Windows
    Define o estilo de edição e navegação no terminal como no cmd tradicional (atalhos com setas, tab, etc.).
    Alternativa: Emacs (para utilizadores avançados/Linux).

  • Set-PSReadLineOption -HistorySearchCursorMovesToEnd
    Quando procuras comandos anteriores com Ctrl+R ou seta para cima, o cursor salta automaticamente para o fim da linha.

  • Set-PSReadLineOption -HistorySaveStyle SaveIncrementally
    Garante que cada comando é gravado no histórico logo após ser executado, mesmo que o terminal feche inesperadamente.

  • Set-PSReadLineOption -HistorySavePath "$env:APPDATA\PowerShell\PSReadLine\History.txt"
    Define o local onde o histórico de comandos é guardado.
    Podes abrir este ficheiro com o VSCode para rever comandos antigos.

5.5. Atalhos e utilitários úteis

Editar e recarregar o perfil:

Set-Alias eprofile "code $PROFILE"
Set-Alias reload ". $PROFILE"

Função para medir tempo de execução:

function Medir-Tempo {
    param([ScriptBlock]$acao)
    Measure-Command $acao
}

Usar assim:

Medir-Tempo { Get-ChildItem -Recurse }

Função para limpar o ecrã e mostrar a pasta atual:

function clsx {
    Clear-Host
    Get-Location
}

Abrir o Chrome em modo incógnito:

function chrome-privado {
    Start-Process "chrome.exe" -ArgumentList "--incognito"
}

Abrir o navegador diretamente numa pesquisa Google

function g {
    param([string]$texto)
    Start-Process "https://www.google.com/search?q=$texto"
}

Ver o tempo atual para Lisboa:

function tempo {
    Invoke-RestMethod wttr.in/Lisboa?format=3
}

Criar e abrir uma nota rápida em Markdown:

function nota-md {
    $ficheiro = "$env:TEMP\nota_$(Get-Date -Format 'yyyyMMdd_HHmmss').md"
    Set-Content $ficheiro "# Nova nota"
    Start-Process $ficheiro
}

Abrir pesquisa no YouTube:

function yt {
    param([string]$termo)
    $url = "https://www.youtube.com/results?search_query=$termo"
    Start-Process $url
}

Ver rapidamente o IP público:

function Meu-IP {
    Invoke-RestMethod "https://ipinfo.io/ip"
}

O que faz o Invoke-RestMethod?

Este comando é usado para fazer chamadas a APIs ou serviços web.

  • Envia pedidos (por exemplo, GET ou POST) para um URL
  • Recebe a resposta e converte automaticamente JSON ou XML em objetos PowerShell
  • É ideal para consumir APIs públicas como previsão do tempo, tradução, IP, etc.

Exemplo:

$dados = Invoke-RestMethod "https://api.agify.io/?name=joao"
Write-Output "Idade estimada: $($dados.age)"

Diferença para Invoke-WebRequest:

  • Invoke-RestMethod → Para dados (JSON/XML)
  • Invoke-WebRequest → Para páginas HTML, download de ficheiros, scraping

5.6. Backup automático do $PROFILE

Garante que não perdes as tuas personalizações:

Copy-Item -Path $PROFILE -Destination "$HOME\Documents\backup_profile.ps1" -Force

5.7. Instalar e usar o Oh My Posh

O oh-my-posh permite personalizar o visual do PowerShell com temas modernos, ícones e cores. Ideal para quem quer um terminal mais bonito e informativo.

Passo 1: Instalar

Se tens o winget, podes instalar com:

winget install JanDeDobbeleer.OhMyPosh -s winget

Podes instalar sem permissões de administrador.

Para garantir compatibilidade com os temas mais recentes, também deves atualizar o módulo PSReadLine:

Install-Module PSReadLine -Scope CurrentUser -Force

Depois de atualizar o PSReadLine é necessário fazer reload do profile . $Profile

Passo 2: Ver onde os temas foram instalados

Por vezes a variável $env:POSH_THEMES_PATH não é criada automaticamente.

Usa este comando para descobrir onde os temas estão:

Get-ChildItem -Path "$env:LOCALAPPDATA\Programs\oh-my-posh" -Recurse -Filter "*.omp.json"

Vais ver uma pasta como:
C:\Users\teu-nome\AppData\Local\Programs\oh-my-posh\themes

Passo 3: Testar um tema

Substitui o caminho abaixo pelo que encontraste no passo anterior:

$tema = "$env:LOCALAPPDATA\Programs\oh-my-posh\themes\paradox.omp.json"
oh-my-posh init pwsh --config $tema | Invoke-Expression

Passo 4: Tornar permanente no $PROFILE

Abre o teu perfil com:

code $PROFILE

E adiciona:

$ompConfig = "$env:LOCALAPPDATA\Programs\oh-my-posh\themes\paradox.omp.json"
oh-my-posh init pwsh --config $ompConfig | Invoke-Expression

Guarda e recarrega com:

. $PROFILE

Passo 5: Visualizar os temas online

Consulta visualmente todos os temas disponíveis aqui:

https://ohmyposh.dev/docs/themes

Passo 6: Instalar uma Nerd Font (opcional)

Para ver os ícones corretamente, instala uma fonte como:

  • Cascadia Code Nerd Font
  • Fira Code Nerd Font

Site oficial: https://www.nerdfonts.com/font-downloads

Depois, muda a fonte no Windows Terminal:

  • Settings → PowerShell → Appearance → Font face

💡 Dica: Se não conseguires usar o oh-my-posh, continua a usar o prompt personalizado manualmente com function prompt { ... }.
