1. O que já aprendeste (Sessão 3)

Antes de continuar, relembra que já aprendemos os seguintes comandos e conceitos na Sessão 3: Objetos, parâmetros, pipeline, output, exportações:

  • Where-Object para filtrar coleções
  • Select-Object para escolher propriedades
  • Sort-Object para ordenar dados
  • Export-Csv, Out-File, Set-Content, Add-Content para gravar ficheiros
  • Format-Table, Format-List para visualizar
  • Uso do pipeline (|) e da variável $_

2. Agrupar dados com Group-Object

Quando queremos saber quantos elementos existem por categoria, usamos Group-Object.

2.1. Exemplo: Agrupar serviços por estado 

Get-Service | Group-Object Status

Este comando agrupa todos os serviços por valor da propriedade Status.
O resultado indica quantos serviços estão em cada estado (Running, Stopped, etc.).

Podes guardar o resultado numa variável e explorar os grupos:

$grupos = Get-Service | Group-Object Status
$grupos[0].Name          # Nome do grupo (ex: "Running")
$grupos[0].Count         # Quantos itens tem esse grupo
$grupos[0].Group         # A lista completa de objetos nesse grupo

Group-Object é muito útil para criar resumos e relatórios estatísticos.

2.2. 🧩 Exercício – Agrupar ficheiros por extensão

  1. Usa Get-ChildItem para obter todos os ficheiros de uma pasta.
  2. Agrupa por extensão (Extension).
  3. Mostra quantos ficheiros existem de cada tipo.

3. Contar e resumir com Measure-Object

O Measure-Object serve para calcular estatísticas simples: número de elementos, somas, médias, mínimo, máximo.

3.1. Exemplo: contar linhas num ficheiro 

Get-Content "ficheiro.txt" | Measure-Object

Resultado:

Count    : 120
Average  :
Sum      :
Maximum  :
Minimum  :
Property :

3.2. Exemplo: média de memória usada por processos 

Get-Process | Measure-Object -Property WorkingSet -Average

O resultado é em bytes — podes dividir por 1MB para converter:

($_.WorkingSet / 1MB)

3.3. 🧩 Exercício – Somar tamanhos de ficheiros

  1. Usa Get-ChildItem para listar os ficheiros de uma pasta.
  2. Soma o tamanho total com Measure-Object -Sum.

4. Filtrar com múltiplas condições

O Where-Object permite usar condições complexas com operadores lógicos como -and, -or, -not.

4.1. Exemplo: Processos com mais de 100 MB e nome começado por “c” 

Get-Process | Where-Object {
    $_.WorkingSet -gt 100MB -and $_.Name -like "c*"
}

4.2. 🧩 Exercício – Filtrar ficheiros grandes e com extensão .log

  1. Usa Get-ChildItem com -Recurse para pesquisar em subpastas.
  2. Filtra com Where-Object ficheiros .log com mais de 1 MB.

5. Exportar dados filtrados

Após filtrar ou transformar dados, podes exportá-los para ficheiros CSV ou TXT.

5.1. Exemplo: Exportar serviços Running para CSV 

Get-Service | Where-Object Status -eq "Running" |
Select-Object Name, Status |
Export-Csv -Path "servicos_ativos.csv" -NoTypeInformation

5.2. 🧩 Exercício – Exportar ficheiros .csv para lista

  1. Lista todos os ficheiros .csv de uma pasta.
  2. Exporta nome e tamanho para ficheiros.csv.

6. Ordenar resultados com Sort-Object

Podes ordenar resultados por qualquer propriedade, em ordem crescente (padrão) ou decrescente.

6.1. Exemplo: Listar os 10 maiores ficheiros 

Get-ChildItem -Path "C:\dados" -Recurse -File |
Sort-Object Length -Descending |
Select-Object Name, Length -First 10

6.2. 🧩 Exercício – Ordenar serviços por nome

  1. Lista todos os serviços.
  2. Ordena alfabeticamente por nome.

7. Resumo visual com Format-Table e Format-List

Estes comandos servem apenas para visualização no terminal (não deves usá-los antes de Export-Csv).

7.1. Exemplo: Visualização em tabela 

Get-Process | Format-Table Name, Id, CPU -AutoSize

7.2. Exemplo: Visualização detalhada 

Get-Service | Format-List *

Usa Select-Object para preparar os dados e Format-* apenas para visualizar no terminal. Para exportar, nunca uses Format-* antes de Export-Csv.

7.3. 🧩 Exercício – Ver os 5 processos com mais CPU num formato limpo

  1. Ordena os processos por CPU.
  2. Mostra apenas os 5 primeiros.
  3. Formata como tabela com autoajuste.
