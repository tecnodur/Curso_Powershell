📦 Sessão 7: Módulos e Bibliotecas Externas


🎯 Objetivo
Aprender a importar, instalar e criar módulos PowerShell para reutilização, partilha e organização de scripts e funções. Vais descobrir como tornar o teu código mais profissional e portátil.

1. O que é um módulo PowerShell?

Um módulo é um ficheiro (ou conjunto de ficheiros) que contém funções, comandos, variáveis ou classes que podem ser reutilizados noutros scripts ou sessões.

Um módulo pode ser:

  • Um único ficheiro .psm1
  • Uma pasta com ficheiros .psm1 e .psd1 (manifesto)
  • Instalado localmente ou via PowerShell Gallery

2. Diferença entre script, função e módulo

Tipo Uso principal Reutilização
Script .ps1 Executa tarefas específicas Limitada
Função Reutilizável dentro de scripts ou sessões Moderada
Módulo Organiza funções para partilha e reutilização Alta

Módulos facilitam a manutenção e partilha do código em projetos maiores ou em equipa.

3. Importar módulos existentes

Listar módulos disponíveis:

Get-Module -ListAvailable

Importar um módulo:

Import-Module NomeDoModulo

Exemplo com módulo nativo:

Import-Module Microsoft.PowerShell.Archive

Ver funções do módulo:

Get-Command -Module Microsoft.PowerShell.Archive

4. Instalar módulos da PowerShell Gallery

Antes de instalar, verifica se tens acesso à Gallery:

Find-Module -Name BurntToast

Instalar o módulo:

Install-Module -Name BurntToast -Scope CurrentUser

Importar:

Import-Module BurntToast

O parâmetro -Scope CurrentUser evita necessidade de permissões de administrador.


Nota

⚠️ Se Find-Module ou Install-Module falharem, pode ser por problemas de rede (sem internet, proxy, firewall) ou permissões impostas pelo sistema.

Alternativa manual (se permitido): acede a https://www.powershellgallery.com/, descarrega o módulo, extrai os ficheiros e coloca-os em:

C:\Users\<teu_utilizador>\Documents\WindowsPowerShell\Modules\NomeDoModulo

Depois, usa Import-Module NomeDoModulo normalmente.

5. Criar o teu próprio módulo

5.1. Estrutura básica

  1. Cria uma pasta com o nome do módulo:

     mkdir C:\PS\MeusModulos\Utilitarios

  2. Dentro da pasta, cria um ficheiro .psm1:

     Set-Content C:\PS\MeusModulos\Utilitarios\Utilitarios.psm1 -Value @'
 function Dizer-Ola {
     param($nome)
     "Olá, $nome!"
 }

 function Somar {
     param($a, $b)
     return $a + $b
 }
 '@

  3. Para usar o módulo:

     Import-Module C:\PS\MeusModulos\Utilitarios\Utilitarios.psm1
 Dizer-Ola "Nuno"
 Somar 10 5

5.2. (Opcional) Criar ficheiro de manifesto .psd1

Podes criar um .psd1 para definir versão, autor, dependências, etc.:

New-ModuleManifest -Path C:\PS\MeusModulos\Utilitarios\Utilitarios.psd1 `
    -RootModule Utilitarios.psm1 `
    -Author "Nuno Durão" `
    -Description "Funções utilitárias de exemplo"

6. Usar módulos em equipa

6.1. Adicionar caminho personalizado à variável de ambiente 

$env:PSModulePath += ";C:\PS\MeusModulos"

Coloca os módulos em subpastas com o mesmo nome do ficheiro .psm1

7. Boas Práticas

  • Usa nomes significativos para módulos e funções
  • Guarda os teus módulos numa pasta organizada (ex: C:\PS\Módulos)
  • Considera criar manifestos .psd1 para maior portabilidade
  • Para partilha em equipa, adiciona o caminho ao $env:PSModulePath
