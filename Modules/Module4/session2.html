🧭 Sessão 2: PowerShell vs CMD: Diferenças e Utilização no Windows

1. O que é o CMD?

O CMD (também conhecido como cmd.exe ou Prompt de Comando) é o terminal tradicional do Windows, baseado na linguagem MS-DOS. Está disponível desde as primeiras versões do Windows e ainda é amplamente usado para tarefas simples de administração.

Para abrir:

cmd

Ou:

Win + R e escrever "cmd"

2. Diferenças entre PowerShell e CMD

Funcionalidade CMD PowerShell
Tipo de linguagem MS-DOS Baseado em objetos .NET
Trabalha com objetos ❌ Texto apenas ✅ Objetos com propriedades e métodos
Suporte a scripts .bat, .cmd .ps1
Exportação de dados Limitada (>, type, findstr) Completa (Export-Csv, Out-File, etc)
Variáveis set x=valor $x = valor
Loops e condições if, goto, for %%x if (...) {}, foreach, while
Comandos externos Suportado Suportado
Personalização Limitada (prompt, %PATH%) $PROFILE, aliases, funções persistentes

3. Exemplos de comandos equivalentes

Tarefa CMD PowerShell
Mostrar diretório atual cd Get-Location
Listar ficheiros dir Get-ChildItem
Copiar ficheiro copy ficheiro.txt destino.txt Copy-Item ficheiro.txt destino.txt
Apagar ficheiro del ficheiro.txt Remove-Item ficheiro.txt
Criar variável set nome=valor $nome = "valor"
Mostrar variável echo %nome% $nome
Comentário em script REM este é um comentário # este é um comentário

⚠️ Nota: Em CMD, as variáveis usam %variavel%. Em PowerShell, usam $variavel.

4. Como criar e correr um script .bat

4.1. Criar o ficheiro

  1. Abre o Bloco de Notas ou outro editor de texto.

  2. Escreve o conteúdo do script, por exemplo:

     @echo off
 echo Olá Mundo!
 pause
  3. Guarda o ficheiro com extensão .bat (ex: meuscript.bat).

4.2. Executar o script

Podes correr o script de várias formas:

  • Clicando duas vezes no ficheiro .bat

  • Escrevendo no terminal cmd:

      meuscript.bat

  • Ou com caminho completo:

      C:\Users\nome\Desktop\meuscript.bat

Scripts .bat funcionam apenas em ambientes cmd.exe. No PowerShell, podem correr, mas com limitações.

5. Casos em que CMD ainda é útil

  • Executar scripts legados ou automações antigas
  • Correr tarefas rápidas com copy, del, echo, pause
  • Simplicidade em ambientes com permissões limitadas
  • Em ambientes onde o PowerShell está desativado por políticas

6. Compatibilidade entre PowerShell e CMD

Podes correr comandos do CMD dentro do PowerShell — e vice-versa com limitações.

6.1. No PowerShell: 

cmd /c dir
cmd /c copy a.txt b.txt

6.2. No CMD: 

powershell -Command "Get-ChildItem"
powershell -File script.ps1

7. Comparação entre ambientes

Tema CMD PowerShell
Suporte a lógica Limitado (goto) Completo (if, foreach, switch)
Trabalhar com ficheiros Simples Robusto
Automatização Scripts .bat básicos Scripts poderosos com lógica e objetos
Manipular texto findstr, type .Replace(), -match, .ToUpper()
Personalização Limitada Avançada com $PROFILE, módulos e funções
Pipelines Texto (|) Objetos reais (|)

8. Como saber qual usar

Se precisas de… Usa…
Script leve, rápido, compatível com qualquer PC cmd
Lógica avançada, objetos, output estruturado PowerShell
Automatizar com robustez e legibilidade PowerShell
Executar scripts antigos ou simples cmd

9. Exercício básico de comparação

9.1. CMD 

@echo off
set nome=Nuno
echo Olá %nome%

9.2. PowerShell 

$nome = "Nuno"
Write-Output "Olá $nome"

10. Resumo final

  • CMD ainda é útil para scripts simples e legados
  • PowerShell oferece muito mais capacidade, especialmente em ambientes modernos
  • Saber ambos é útil, mas PowerShell é o futuro da automação no Windows
