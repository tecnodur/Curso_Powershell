1. O que são Serviços no Windows?

Serviços são aplicações ou componentes que funcionam em segundo plano, muitas vezes sem qualquer interface visível, e são iniciados automaticamente com o sistema operativo ou sob determinadas condições.

1.1. Para que servem?

Serviços são usados para manter funcionalidades essenciais do sistema e de aplicações, mesmo quando nenhum utilizador está ligado.

Exemplos comuns:

  • Spooler de Impressão
    Gere a fila de impressão para impressoras ligadas ao computador.

  • Windows Update
    Verifica e instala atualizações automaticamente.

  • Serviço de Hora do Windows
    Mantém o relógio do sistema sincronizado com servidores de hora.

  • Serviço de Licenciamento
    Garante que as licenças de software estão válidas e ativas.

1.2. Características dos serviços

  • Podem ser automáticos, manuais ou desativados
  • Podem estar em execução, parados ou em pausa
  • Correm sob contas de sistema com permissões específicas
  • São geridos através do Serviços.msc ou com PowerShell

1.3. Como ver os serviços instalados

Com PowerShell:

Get-Service

Exemplo de resultado:

Status   Name               DisplayName
------   ----               -----------
Running  Spooler            Print Spooler
Stopped  WSearch            Windows Search
Running  WinDefend          Windows Defender Antivirus Service

Para obter mais informação sobre um serviço

Get-CimInstance -ClassName Win32_Service -Filter "Name='Spooler'" | Format-List Name, DisplayName, State, StartMode, PathName, Description

Substituir Spooler pelo nome do serviço que quisermos.

Saber lidar com serviços é essencial para diagnosticar problemas, automatizar tarefas e garantir que aplicações funcionam corretamente no ambiente Windows.

Para abrir o Visualizador de Eventos: 

eventvwr.msc

2. Gerir Serviços com PowerShell

2.1. Comandos principais

  • Get-Service – Lista os serviços e o seu estado atual
  • Start-Service – Inicia um serviço
  • Stop-Service – Para um serviço
  • Restart-Service – Reinicia um serviço

2.2. Exemplos práticos

  • Listar todos os serviços:

      Get-Service

  • Ver o estado de um serviço específico:

      Get-Service -Name "Spooler"

  • Iniciar o serviço de spooler de impressão:

      Start-Service -Name "Spooler"

  • Parar o serviço de spooler de impressão:

      Stop-Service -Name "Spooler"

  • Reiniciar o serviço de spooler de impressão:

      Restart-Service -Name "Spooler"

⚠️ Alguns comandos podem necessitar de privilégios de administrador.

3. Diferença entre Get-EventLog e Get-WinEvent

PowerShell tem dois comandos para consultar logs de eventos. A escolha depende do tipo de log e do nível de detalhe que precisas.

Característica Get-EventLog Get-WinEvent
Logs suportados Apenas logs clássicos (Application, System, Security) Logs clássicos e modernos (XML)
Introduzido em PowerShell 1.0 PowerShell 2.0
Desempenho Mais lento com muitos eventos Mais rápido e eficiente
Filtros Limitados (por tipo, ID, etc) Suporta filtros avançados com Hashtable e XPath
Acesso a logs avançados ❌ ✅ Inclui logs como Microsoft-Windows-*
Estado atual Mantido por compatibilidade Recomendado para novos scripts

O que são logs modernos?

  • Foram introduzidos no Windows Vista e versões seguintes
  • Usam formato XML e têm canais próprios por aplicação
  • Só estão acessíveis com Get-WinEvent

Exemplos de logs modernos:

  • Microsoft-Windows-PowerShell/Operational
  • Microsoft-Windows-TaskScheduler/Operational
  • Microsoft-Windows-Windows Defender/Operational

Recomendação

  • Para scripts novos, usa Get-WinEvent sempre que possível
  • Usa Get-EventLog apenas se estiveres a trabalhar com logs legados ou em sistemas antigos

4. Consultar Logs de Eventos do Windows

4.1. Usar Get-EventLog

  • Listar os logs disponíveis:

      Get-EventLog -List

  • Obter as 5 entradas mais recentes do log de sistema:

      Get-EventLog -LogName System -Newest 5

  • Filtrar eventos de erro no log de sistema:

      Get-EventLog -LogName System -EntryType Error

  • Procurar eventos por ID específico:

      Get-EventLog -LogName Application | Where-Object {$_.EventID -eq 1000}

4.2. Usar Get-WinEvent para logs modernos

  • Obter eventos recentes do log de Aplicação:

      Get-WinEvent -LogName Application -MaxEvents 5

  • Filtrar eventos:

      Get-WinEvent -LogName System -MaxEvents 10 | Format-Table TimeCreated, Id, LevelDisplayName, Message -AutoSize

Get-WinEvent é mais eficiente e suporta logs modernos introduzidos no Windows Vista e posteriores.

5. Criar Alertas com PowerShell

5.1. Exibir uma mensagem popup simples 

$wshell = New-Object -ComObject Wscript.Shell
$wshell.Popup("Operação concluída com sucesso!", 0, "Alerta", 0x1)

5.2. Usar System.Windows.Forms para mensagens mais avançadas 

Add-Type -AssemblyName System.Windows.Forms
[System.Windows.Forms.MessageBox]::Show("Deseja continuar?", "Confirmação", [System.Windows.Forms.MessageBoxButtons]::YesNo, [System.Windows.Forms.MessageBoxIcon]::Question)

Estas mensagens podem ser úteis para interações simples com o utilizador ou notificações.

6.3. Criar Alerta Automático para Monitorizar Serviços

Podes usar o seguinte script como uma tarefa agendada que corre a cada X minutos. Ele verifica se um serviço específico está parado e mostra um alerta no ecrã se isso acontecer.

Exemplo com o serviço Spooler:

Código 
$servico = Get-Service -Name "Spooler"
if ($servico.Status -ne "Running") {
    Add-Type -AssemblyName System.Windows.Forms
    [System.Windows.Forms.MessageBox]::Show("⚠️ O serviço Spooler está parado!", "Alerta de Serviço", [System.Windows.Forms.MessageBoxButtons]::OK, [System.Windows.Forms.MessageBoxIcon]::Warning)
}

Como usar:

  1. Guarda o script como verificar_servico.ps1
  2. Cria uma tarefa agendada no Windows:
    • Ação: correr powershell.exe com o argumento "C:\Scripts\verificar_servico.ps1"
    • Frequência: por exemplo, a cada 10 minutos
  3. Verifica que o PowerShell tem permissões para mostrar mensagens popup

Este método permite criar alertas visuais automáticos sem depender de software externo.

7. Boas Práticas

  • Privilégios: Executa o PowerShell como administrador para gerir serviços e aceder a todos os logs.
  • Filtragem: Usa parâmetros como -After, -EntryType e -EventID para filtrar logs e obter informações relevantes.
  • Automatização: Combina comandos para criar scripts que monitorizam serviços e logs automaticamente.
  • Notificações: Utiliza mensagens popup para informar o utilizador sobre o estado das operações.
