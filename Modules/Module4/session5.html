1. Revisão Rápida: Exportações e Notificações

1.1. Exportar para CSV (já visto) 

Get-Process | Export-Csv -Path "processos.csv" -NoTypeInformation

1.2. Notificação visual simples (já vista) 

$wshell = New-Object -ComObject Wscript.Shell
$wshell.Popup("Operação concluída com sucesso!", 0, "Alerta", 0x1)

Nesta sessão vamos aplicar esses conceitos a situações práticas e adicionar novas técnicas, como exportar para HTML e enviar emails.

2. Exportar para HTML

PowerShell permite gerar ficheiros .html facilmente a partir de tabelas de objetos.

2.1. Exemplo: 

Get-Service | Select-Object Name, Status, StartType |
ConvertTo-Html -Title "Relatório de Serviços" |
Out-File -FilePath "servicos.html"

2.2. Como funciona ConvertTo-Html

O comando ConvertTo-Html transforma uma tabela de objetos PowerShell num ficheiro HTML com estrutura de <table>.

Utilização básica 

Get-Process | ConvertTo-Html | Out-File "relatorio.html"

Este comando:

  • Cria uma tabela HTML com os dados de Get-Process
  • Usa os nomes das propriedades como cabeçalhos de coluna

🔧 Personalizações úteis

  • -Title "Título da Página"
    Define o título do documento HTML:

    ConvertTo-Html -Title "Relatório de Processos"

  • -PreContent, -PostContent
    Permitem adicionar texto antes ou depois da tabela:

    ConvertTo-Html -PreContent "<h1>Relatório</h1>"

  • -Property
    Especifica quais colunas mostrar:

    Get-Service | ConvertTo-Html -Property Name, Status

Estilizar com CSS

Podes adicionar estilo simples assim:

$css = "<style>table {border-collapse:collapse} td,th {border:1px solid black;padding:4px}</style>"
$html = Get-Service | ConvertTo-Html -Property Name, Status -Head $css -Title "Serviços"
$html | Out-File "servicos.html"

O parâmetro -Head insere conteúdo dentro da <head> da página, ideal para CSS.

Nota final

  • ConvertTo-Html não abre o browser automaticamente

  • Se quiseres ver o resultado, usa:

    Invoke-Item "servicos.html"

Ou abre o ficheiro manualmente com um browser.

3. Enviar Email com PowerShell

É possível enviar emails automaticamente com PowerShell de duas formas principais: via SMTP ou via Outlook instalado.

3.1. Enviar Email via SMTP (ex: Gmail)

Usa o comando Send-MailMessage, que requer um servidor SMTP e credenciais válidas.

$cred = Get-Credential  # Usa o teu email e uma App Password
Send-MailMessage -From "teu@email.com" -To "destino@email.com" `
-Subject "Relatório" -Body "Segue em anexo o relatório." `
-SmtpServer "smtp.gmail.com" -Port 587 -UseSsl -Credential $cred `
-Attachments "C:\Relatorios\relatorio.html"

⚠️ Nota: Para enviares emails com o Gmail, precisas de ativar a autenticação em dois fatores e gerar uma App Password.
Visita myaccount.google.com/apppasswords.

3.2. Enviar Email via Outlook (instalado localmente)

Se tens o Outlook instalado e configurado, podes enviar emails sem precisar de SMTP nem credenciais:

$outlook = New-Object -ComObject Outlook.Application
$mail = $outlook.CreateItem(0)

$mail.To = "destinatario@email.com"
$mail.Subject = "Relatório de Erros"
$mail.Body = "Segue em anexo o relatório."
$mail.Attachments.Add("C:\Relatorios\erros_20240524_1400.csv")

$mail.Send()

O Outlook deve estar instalado e associado a uma conta válida. Pode surgir um aviso de segurança ao enviar.

4. Exercícios

4.1. 🧩 Exportar lista de processos ativos para HTML

Desafio: Cria um script que:

  • Obtém a lista de processos
  • Seleciona apenas os campos Nome, ID e Memória
  • Gera um relatório em HTML com título “Relatório de Processos”
  • Guarda em processos.html

4.2. 🧩 Criar notificação visual após exportação

Desafio: Altera o exercício anterior para mostrar uma notificação quando o relatório for guardado com sucesso.

4.3. 🧩 Guardar logs de erros num CSV com timestamp

Desafio: Cria um script que:

  • Filtra eventos de erro do log “System” das últimas 24h
  • Exporta para CSV
  • Nome do ficheiro deve incluir a data e hora da exportação

4.4. 🧩 Enviar email com relatório (extra)

Desafio: Envia por email o ficheiro gerado no exercício 4.3.

Este exercício só deve ser feito se tiveres servidor SMTP funcional ou conta configurada (ex: Gmail com app password).

5. Boas Práticas

  • Usa Out-File com codificação UTF-8 se o ficheiro for HTML com acentuação
  • Usa -NoTypeInformation ao exportar CSVs
  • Garante que os caminhos de ficheiro existem (Test-Path, New-Item -ItemType Directory)
  • Se estiveres a partilhar ficheiros com outros, evita caminhos locais como C:\Users\nome
