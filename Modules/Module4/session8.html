1. Depuração e Diagnóstico

Durante o desenvolvimento de scripts, é essencial conseguir ver o que está a acontecer nos bastidores. PowerShell inclui ferramentas como mensagens de depuração, mensagens informativas e a capacidade de gravar tudo o que acontece no terminal.

1.1. Write-Verbose, Write-Debug e Write-Information

Estes comandos servem para mostrar mensagens internas do script, úteis para compreender o que está a acontecer durante a execução.

Por defeito, estas mensagens só aparecem se usares os parâmetros -Verbose ou -Debug ao executar a função.

Exemplo:

function Testar-Depuracao {
    [CmdletBinding()]
    param()

    Write-Verbose "A iniciar cálculo"
    Write-Debug "Preparar variável"
    $resultado = 10 * 2
    Write-Output "Resultado final: $resultado"
}

Testar-Depuracao -Verbose

Neste exemplo, só vais ver a mensagem Write-Verbose se usares o parâmetro -Verbose. A mensagem Write-Debug só aparece se também adicionares -DebugPreference ‘Continue’.

1.2. Start-Transcript

Este comando grava tudo o que aparece no terminal (incluindo o que escreves e o que o script devolve) num ficheiro de texto. É útil para auditoria ou para rever execuções anteriores.

Exemplo:

Start-Transcript -Path "log_execucao.txt"
Write-Output "Este texto vai ficar gravado no ficheiro"
Stop-Transcript

2. Funções Avançadas

Funções são blocos reutilizáveis de código. Em PowerShell, podemos torná-las avançadas ao usar CmdletBinding, o que permite funcionalidades como validação automática de parâmetros, suporte a -Verbose, -ErrorAction, e entrada por pipeline.

Exemplo completo:

function Get-Mensagem {
    [CmdletBinding()]
    param(
        [Parameter(Mandatory)]           # obriga a fornecer o parâmetro
        [ValidateSet("PT", "EN")]        # limita as opções possíveis
        [string]$Idioma                  # define o tipo de dado
    )

    switch ($Idioma) {
        "PT" { "Olá mundo" }
        "EN" { "Hello world" }
    }
}

Se tentares correr esta função sem o parâmetro -Idioma, o PowerShell vai perguntar-te qual o valor. E se escreveres algo fora do que está definido no ValidateSet, ele vai dar erro.

Uso:

Get-Mensagem -Idioma PT

Esta estrutura ajuda a evitar erros, torna os scripts mais fáceis de manter e melhora a documentação automática com Get-Help.

3. Integração com APIs REST e JSON

PowerShell pode interagir com serviços web, como APIs REST, usando comandos HTTP. Isto permite obter dados de serviços externos ou integrar sistemas de forma simples e direta.

3.1. Invoke-RestMethod

Este comando envia um pedido HTTP e devolve a resposta já convertida em objeto PowerShell.

Exemplo Vost (Voluntários Digitais em Situações de Emergência):

$ocorrencias = Invoke-RestMethod "https://api.vost.pt/bd/ocorrencias.json"
$filtro = "Setúbal"  # Podes mudar para qualquer distrito
$filtradas = $ocorrencias | Where-Object { $_.distrito -eq $filtro }
$filtradas | Select-Object titulo, concelho, data_hora | Format-Table -AutoSize

Exemplo IPMA:

$codigoCidade = 1110600
$url = "https://api.ipma.pt/open-data/forecast/meteorology/cities/daily/$codigoCidade.json"
$previsao = Invoke-RestMethod $url
$previsao.data | ForEach-Object {
    [PSCustomObject]@{
        Data = $_.forecastDate
        Min = $_.tMin
        Max = $_.tMax
        Tempo = $_.predWindDir
    }
} | Format-Table -AutoSize

3.2. Conversão de JSON

Para trabalhar com JSON no PowerShell:

$json = '{"nome":"Nuno","idade":42}'
$objeto = $json | ConvertFrom-Json
$objeto.nome

Para exportar:

$objeto | ConvertTo-Json

Estas funcionalidades são ideais para integrar aplicações web, consultar bases de dados online, ou criar dashboards automatizados.

4. Leitura de Feeds RSS com PowerShell

Feeds RSS são uma forma simples de obter atualizações de notícias, blogs, sites institucionais ou mesmo alertas. O PowerShell consegue consumir estes feeds diretamente, tratando o conteúdo como XML estruturado.

4.1. Como funciona

Em vez de Invoke-RestMethod, usamos Invoke-WebRequest com controlo explícito da codificação UTF-8 para garantir que os caracteres acentuados aparecem corretamente.

Exemplo Noticias ao Minuto:

# URL do feed RSS de últimas notícias do Notícias ao Minuto
$url = "https://www.noticiasaominuto.com/rss/ultima-hora"

# Faz o pedido HTTP com cabeçalho para garantir que a codificação vem em UTF-8
$response = Invoke-WebRequest -Uri $url -Headers @{ "Accept-Charset" = "utf-8" }

# Converte manualmente o conteúdo da resposta em UTF-8 e interpreta como XML
[xml]$rss = [System.Text.Encoding]::UTF8.GetString($response.RawContentStream.ToArray())

# Extrai a lista de itens (notícias) do feed
$items = $rss.rss.channel.item

# Garante que mesmo que haja só uma notícia, a variável $items será sempre um array
if ($items -isnot [System.Array]) { $items = @($items) }

# Seleciona as 5 primeiras notícias e mostra o título, link e data
$items | Select-Object -First 5 `
    @{Name = "Título"; Expression = {
        # Se o título estiver dentro de CDATA, extrai-o corretamente
        if ($_.title.'#cdata-section') { $_.title.'#cdata-section' }
        else { $_.title }
    }},
    @{Name = "Link"; Expression = { $_.link }},
    @{Name = "Data"; Expression = { $_.pubDate }} |
    Format-Table -AutoSize

4.2. Filtrar por palavra-chave

Podes aplicar filtros à lista de itens para ver só notícias que contenham uma palavra no título, por exemplo “Portugal”:

$items | Where-Object {
    if ($_.title.'#cdata-section') {
        $_.title.'#cdata-section' -like "*Portugal*"
    } else {
        $_.title -like "*Portugal*"
    }
}

4.3. Exportar para CSV

Também podes exportar as notícias filtradas ou todas para um ficheiro .csv:

$items | Select-Object `
    @{Name = "Título"; Expression = {
        if ($_.title.'#cdata-section') { $_.title.'#cdata-section' }
        else { $_.title }
    }},
    @{Name = "Link"; Expression = { $_.link }},
    @{Name = "Data"; Expression = { $_.pubDate }} |
    Export-Csv -Path "noticias.csv" -NoTypeInformation -Encoding UTF8

4.4. Exemplo Publico com Regex: 

$url = "https://feeds.feedburner.com/PublicoRSS"
$response = Invoke-WebRequest -Uri $url
$content = $response.Content

# Extrai blocos <item>...</item>
$items = [regex]::Matches($content, "<item>(.*?)</item>", "Singleline")

foreach ($match in $items) {
    $item = $match.Groups[1].Value

    $title = [regex]::Match($item, "<title>(.*?)</title>").Groups[1].Value
    $link = [regex]::Match($item, "<link>(.*?)</link>").Groups[1].Value
    $pubDate = [regex]::Match($item, "<pubDate>(.*?)</pubDate>").Groups[1].Value

    [PSCustomObject]@{
        Título = $title
        Link = $link
        Data = $pubDate
    }
}

5. Execução Assíncrona com Jobs

Por vezes, queremos correr tarefas em segundo plano enquanto o script continua. Os jobs em PowerShell permitem isso. Cada job corre numa sessão separada, sem bloquear o terminal principal.

5.1. Criar e gerir jobs

Exemplo simples:

$job = Start-Job -ScriptBlock {
    Start-Sleep -Seconds 5
    Get-Date
}

Wait-Job $job
$resultado = Receive-Job $job
Remove-Job $job

Podes também correr vários jobs ao mesmo tempo:

$urls = @("https://www.google.com", "https://www.microsoft.com")
$jobs = @()

foreach ($url in $urls) {
    $jobs += Start-Job -ScriptBlock {
        param($link)
        try {
            Invoke-WebRequest -Uri $link -UseBasicParsing -TimeoutSec 5
            "$link está online"
        } catch {
            "$link está offline"
        }
    } -ArgumentList $url
}

$jobs | Wait-Job
$jobs | ForEach-Object { Receive-Job $_; Remove-Job $_ }

Isto permite fazer chamadas a múltiplos sites ou APIs em simultâneo, poupando tempo em execuções longas.

6. Exercício: Verificação de URLs com Logging e Exportação

Este exercício junta vários dos conceitos abordados nesta sessão: leitura de ficheiros, execução em paralelo com jobs, chamadas HTTP a APIs, exportação de resultados para JSON e gravação de logs com Start-Transcript.

6.1. Objetivo

Criar um script que:

  • Lê uma lista de URLs a partir de um ficheiro urls.txt
  • Usa jobs para verificar cada URL em paralelo
  • Guarda o resultado de cada chamada (com sucesso ou erro)
  • Exporta os resultados num ficheiro resultado.json
  • Grava todo o processo num ficheiro de log

6.2. Explicação

Vamos usar:

  • Get-Content para ler o ficheiro com as URLs
  • Start-Job para criar jobs paralelos
  • Invoke-WebRequest para testar os links
  • [PSCustomObject] para criar objetos com os resultados
  • ConvertTo-Json e Set-Content para exportar para JSON
  • Start-Transcript para registar tudo o que aparece no terminal

6.3. Solução Sugerida 

$urls = Get-Content "urls.txt"
$jobs = @()
$resultados = @()

Start-Transcript -Path "log_execucao.txt"

foreach ($url in $urls) {
    $jobs += Start-Job -ScriptBlock {
        param($link)
        try {
            $res = Invoke-WebRequest -Uri $link -UseBasicParsing -TimeoutSec 5
            [PSCustomObject]@{
                URL = $link
                Status = $res.StatusCode
                Sucesso = $true
            }
        } catch {
            [PSCustomObject]@{
                URL = $link
                Status = "Erro"
                Sucesso = $false
            }
        }
    } -ArgumentList $url
}

$jobs | Wait-Job
$resultados = $jobs | ForEach-Object { Receive-Job $_; Remove-Job $_ }

$resultados | ConvertTo-Json | Set-Content "resultado.json"

Stop-Transcript

O ficheiro resultado.json vai conter um array de objetos com o estado de cada URL. O ficheiro log_execucao.txt terá tudo o que foi escrito no terminal durante a execução.

7. Usar o ChatGPT para Construir Scripts

O ChatGPT pode ser uma ferramenta poderosa para ajudar a escrever, corrigir ou melhorar scripts PowerShell. Não substitui o conhecimento, mas acelera a aprendizagem e o desenvolvimento de soluções.

7.1. Exemplos de uso

Gerar a estrutura base de um script

Podes pedir algo como:

“Cria um script PowerShell que leia um ficheiro CSV com nomes e emails, e envie um email personalizado para cada linha usando Send-MailMessage.”

Explicar um erro ou linha de código

“O que significa este erro em PowerShell: ‘The term is not recognized as the name of a cmdlet’?”

“Explica o que faz esta linha: Get-Process | Where-Object { $_.CPU -gt 100 }”

Otimizar código existente

“Este script está muito lento. Há forma de o tornar mais rápido?”

“Consegue-se fazer o mesmo com menos linhas?”

7.2. Boas práticas ao usar ChatGPT

  • Explica o contexto do que queres automatizar
  • Usa linguagem simples e direta
  • Revisa sempre o código sugerido antes de executar
  • Testa por partes, especialmente se o script for longo
  • Usa-o como apoio, mas garante que entendes o que o script faz
