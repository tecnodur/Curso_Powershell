1. Introdução às Estruturas Condicionais

Num script, as estruturas condicionais permitem tomar decisões com base em valores ou situações.
Em vez de o código correr sempre da mesma forma, conseguimos alterar o comportamento dependendo de condições específicas.

Por exemplo:

  • Se a idade do utilizador for maior ou igual a 18, pode votar.
  • Se o ficheiro for .csv, deve ser processado de forma diferente de um .txt.
  • Se o login falhar três vezes, o acesso é bloqueado.

💬 Pensar como “Se… então…”

Ao programar, estamos constantemente a decidir:

“Se isto for verdade, então faz aquilo.”

O PowerShell usa o comando if para representar estas decisões.

Exemplo visual simples:

$idade = 20

if ($idade -ge 18) {
    Write-Output "Podes votar."
}

2. Sintaxe básica do if

O comando if permite que um bloco de código seja executado apenas quando uma determinada condição é verdadeira.

A estrutura básica é:

if (<condição>) {
    # Código a executar se a condição for verdadeira
}
  • A condição tem de resultar em $true ou $false.
  • As chavetas {} são obrigatórias, mesmo que o bloco tenha apenas uma linha.

Exemplo simples:

$temperatura = 30

if ($temperatura -gt 25) {
    Write-Output "Está calor hoje!"
}

Neste exemplo:

  • A condição $temperatura -gt 25 avalia se a temperatura é superior a 25.
  • Se for verdadeira, a mensagem “Está calor hoje!” será mostrada.

Notas importantes:

  • Podes usar qualquer expressão que devolva $true ou $false.
  • O código dentro do if só é executado se a condição for verdadeira.
  • Se for falsa, o PowerShell simplesmente ignora o bloco de chavetas {}.

3. Operadores de comparação e outros operadores (revisão)

Antes de criarmos decisões nos nossos scripts, é importante dominar os principais operadores de comparação, lógicos e de verificação de dados.

3.1. Operadores de comparação

Os operadores de comparação devolvem sempre $true ou $false e permitem comparar números, texto, datas ou outros objetos.

Operador Significado Case-sensitive Exemplo Resultado
-eq Igual Não "ana" -eq "Ana" $true
-ceq Igual Sim "ana" -ceq "Ana" $false
-ne Diferente Não 5 -ne 3 $true
-cne Diferente Sim "a" -cne "A" $true
-gt Maior que Não 7 -gt 4 $true
-cgt Maior que Sim (comparação de strings) Depende
-lt Menor que Não 3 -lt 8 $true
-clt Menor que Sim (comparação de strings) Depende
-ge Maior ou igual Não 5 -ge 5 $true
-cge Maior ou igual Sim (comparação de strings) Depende
-le Menor ou igual Não 2 -le 3 $true
-cle Menor ou igual Sim (comparação de strings) Depende

⚠️ Nota sobre comparação de texto

Quando comparas strings com operadores como -gt, -lt, -ge, -le, o PowerShell faz a comparação por ordem alfabética (lexicográfica).

  • A comparação é case-insensitive por omissão.
  • "Ana" é considerado igual a "ana" para efeitos de comparação.
  • Para distinguir maiúsculas de minúsculas, usa operadores como -cgt, -clt, -ceq.

Exemplo:

"zebra" -gt "abelha"   # $true
"Ana" -ceq "ana"       # $false

3.2. Operadores lógicos

Permitem combinar várias condições, tornando as decisões mais poderosas.

Operador Descrição Exemplo Resultado
-and E lógico $x -gt 5 -and $y -lt 10 $true se ambas forem verdadeiras
-or OU lógico $x -eq 10 -or $y -eq 20 $true se uma for verdadeira
-not Negação lógica -not ($x -eq 5) Inverte o resultado

Exemplo prático:

$idade = 25
$nacionalidade = "Portugal"

if (($idade -ge 18) -and ($nacionalidade -eq "Portugal")) {
    Write-Output "Autorizado a votar."
}

3.3. Operadores de tipo

Permitem verificar se um valor é de um determinado tipo de dados.

Operador Descrição Exemplo Resultado
-is Testa se é de um tipo $var -is [int] $true ou $false

Exemplo prático:

$numero = 42

if ($numero -is [int]) {
    Write-Output "É um número inteiro."
}

3.4. Operadores de correspondência de padrões

Usados para trabalhar com texto baseado em padrões ou expressões regulares.

Operador Descrição Exemplo Resultado
-like Corresponde a padrão (wildcards) "Ana" -like "A*" $true
-match Corresponde a expressão regular "ana123" -match "\d{3}" $true
-contains Verifica se uma coleção contém um valor @(1,2,3) -contains 2 $true
-in Verifica se um valor está presente numa coleção "banana" -in $frutas $true

Exemplo prático:

$nome = "Ana Clara"

if ($nome -like "Ana*") {
    Write-Output "O nome começa por Ana."
}

4. Inputs e Expressões Regulares

Antes de avançarmos no controlo de fluxos (if, else, elseif), vamos fazer uma pequena pausa estratégica.

Nesta secção, vais aprender duas ferramentas fundamentais para tornar os teus scripts mais dinâmicos:

  • Como pedir informações ao utilizador (Read-Host).
  • Como validar padrões de texto usando expressões regulares (-match).

Estas técnicas vão ser muito úteis para os exercícios práticos que vêm a seguir e para criar scripts mais interativos e inteligentes.

4.1. Pedir input ao utilizador (Read-Host)

Em scripts mais dinâmicos, muitas vezes queremos perguntar algo ao utilizador e reagir de acordo com a resposta.

Em PowerShell usamos o comando Read-Host.

Sintaxe básica:

$variavel = Read-Host "Mensagem para o utilizador"

Exemplo:

$nome = Read-Host "Qual é o teu nome?"
Write-Output "Olá, $nome!"

4.2. Introdução às Expressões Regulares (-match)

As expressões regulares (ou regex) são uma maneira de procurar padrões em texto.
São muito úteis para validar inputs, procurar formatos específicos, extrair partes de texto, entre outros.

O que é uma expressão regular?

É uma sequência de símbolos especiais que define um padrão.
Por exemplo:

  • "^\d+$" significa “um ou mais dígitos” (números inteiros).
  • "^A.*" significa “começa por A”.
  • "^\w+@\w+\.\w+$" pode validar emails simples.

Em PowerShell, usamos o operador -match para aplicar expressões regulares. Por padrão, este operador é case-insensitive (ignora maiúsculas e minúsculas).
Se quiseres uma correspondência sensível a maiúsculas/minúsculas, usa o operador -cmatch.
Também podes usar o modificador inline (?-i) dentro da regex, por exemplo: -match "(?-i)^[A-Z]".

Regras básicas de expressões regulares

Símbolo Significado Exemplo
. Qualquer caractere "a", "1", "%"
* Zero ou mais repetições "ab*" → "a", "ab", "abb"
+ Uma ou mais repetições "ab+" → "ab", "abb", não "a"
? Zero ou uma repetição "ab?" → "a", "ab"
\d Um dígito (0–9) "123"
\w Letra, número ou underscore "abc_123"
^ Começo do texto "^Ana" → começa por Ana
$ Fim do texto "ana$" → termina em ana
(?-i) Torna a regex case-sensitive "Ana" -match "(?-i)^A"

Como construir uma expressão regular simples

Agora que já conheces os símbolos básicos, vamos ver como os combinar para criar padrões mais úteis.

1. Pensar no que queremos detetar

Antes de escrever a regex, faz estas perguntas:

  • A palavra tem letras? Números? Ambos?
  • Há limite no número de caracteres?
  • Tem de começar ou terminar de uma certa forma?

2. Combinando os símbolos

  • Para letras, números e underscores usamos \w.
  • Para números apenas, usamos \d.
  • Para especificar quantidade, usamos +, *, {n}, {n,m}, etc.
  • Para indicar início ou fim do texto, usamos ^ e $.

Exemplo com -match (ignora maiúsculas/minúsculas)

  1. Pede ao utilizador um nome.
  2. Verifica se começa por “a” ou “A”.

  3. Mostra uma mensagem se corresponder.

     $nome = Read-Host "Introduz o teu nome"

 if ($nome -match "^a") {
     Write-Output "maiúsculas/minúsculas"
 }

Exemplo com -cmatch (distingue maiúsculas/minúsculas)

  1. Pede ao utilizador um nome.
  2. Verifica se começa por “A” maiúsculo exatamente.

  3. Mostra uma mensagem se corresponder.

     $nome = Read-Host "Introduz o teu nome"

 if ($nome -cmatch "^A") {
     Write-Output "Começa por A maiúsculo (case-sensitive)"
 }

Exemplo com modificador inline (?-i) (distingue maiúsculas/minúsculas)

  1. Pede ao utilizador um nome.
  2. Verifica se começa por “A” maiúsculo exatamente.

  3. Mostra uma mensagem se corresponder.

     $nome = Read-Host "Introduz o teu nome"

 if ($nome -match "(?-i)^A") {
     Write-Output "Começa por A maiúsculo (case-sensitive)"
 }

Sobre o uso de ^ e $

  • O símbolo ^ indica o início da string.
  • O símbolo $ indica o fim da string.

Se quiseres apenas verificar se existe um padrão em qualquer parte da string, não precisas de usar ^ nem $.

Exemplo:

"Ficheiro123.txt" -match "\d+"      # True (encontra o "123")
"Ficheiro123.txt" -match "^\d+$"    # False (não é só um número)

E se quiseres encontrar todas as ocorrências?

O operador -match apenas devolve:

  • $true se encontrar uma correspondência,
  • E guarda apenas a primeira em $matches[0].

Se precisares de todas as ocorrências (global match), usa:

$resultado = [regex]::Matches($texto, $padrão)

Exemplo:

$texto = "Ana1, Teste9, _valor2"
$regex = "\w+\d"
$matches = [regex]::Matches($texto, $regex)

foreach ($m in $matches) {
    Write-Output $m.Value
}

Isto vai devolver todas as palavras que terminam em número.

💡 Dica: No site regex101.com, ativa a opção g (global) para veres todos os matches — por defeito, só mostra o primeiro.

3. Exemplos práticos

Detectar palavras como egyt1, _sdag4rg, Ana

Padrão: “Um ou mais caracteres que sejam letras, números ou underscore”

Expressão regular:

^\w+$

Explicação:

  • ^ ➔ início do texto
  • \w+ ➔ uma ou mais letras, números ou underscores
  • $ ➔ fim do texto

Detectar palavras que terminam em número

Exemplo: Ana1, Teste9, _valor2

Expressão regular:

^\w+\d$

Explicação:

  • ^\w+ ➔ começa com uma ou mais letras/números/underscores
  • \d ➔ termina com um único dígito
  • $ ➔ fim do texto

Detectar uma palavra com exatamente 4 letras seguidas de 1 número

Exemplo: abcd1, test9

Expressão regular:

^\w{4}\d$

Explicação:

  • ^\w{4} ➔ quatro caracteres que podem ser letras, números ou underscores
  • \d ➔ um único dígito
  • $ ➔ fim do texto

Dicas para começar

  • Experimenta padrões simples primeiro.
  • Testa exemplos reais no PowerShell com -match.
  • Vai adicionando complexidade aos poucos.

Onde praticar Expressões Regulares?

Podes aprender e construir expressões regulares utilizando a documentação oficial da Microsoft:

🔗 Quick Reference – Regular Expression Language (Microsoft) 🔗 about_Comparison_Operators (PowerShell)

Podes testar expressões regulares compatíveis com PowerShell no site regex101.com, escolhendo a opção .NET 7.0 (C#) como linguagem.

4.3. 🧩 Exercícios práticos – Inputs e Comparações (Só if)

Antes de avançarmos para else, vamos praticar apenas com if.

🧩 Exercício 1 – Validação de nome

  1. Pede ao utilizador para introduzir um nome (Read-Host).
  2. Verifica se o nome começa pela letra “A” (maiúscula ou minúscula).
  3. Se começar por A, mostra a mensagem “Nome começa por A”.

    Podes usar o operador -match com o padrão (?i)^a.

Resposta 
$nome = Read-Host "Introduz o teu nome"

if ($nome -match '^(?i)a') {
    Write-Output "Nome começa por A"
}

🧩 Exercício 2 – Validação de número

  1. Pede ao utilizador para introduzir um número (Read-Host).
  2. Confirma se é um número usando a expressão regular ^\d+$.
  3. Se for número, mostra “Número válido”.
Resposta 
$valor = Read-Host "Introduz um número"

if ($valor -match '^\d+$') {
    Write-Output "Número válido"
}

5. Estruturas Condicionais: else e elseif

Até agora já vimos como usar o if para executar código quando uma condição é verdadeira.

Agora vamos aprender a tratar:

  • o que acontece se a condição for falsa (else);
  • como testar várias condições diferentes (elseif).

5.1. O comando else

O else permite executar um bloco de código alternativo caso a condição do if seja falsa ($false).

Sintaxe: 

if (condição) {
    # Código se a condição for verdadeira
}
else {
    # Código se a condição for falsa
}

Exemplo: 

$idade = 16

if ($idade -ge 18) {
    Write-Output "És maior de idade."
}
else {
    Write-Output "És menor de idade."
}

Neste caso, como $idade é 16, a condição é falsa e executa-se o bloco do else.

5.2. O comando elseif

O elseif permite testar várias condições diferentes, uma a seguir à outra.

O PowerShell avalia cada condição pela ordem em que aparecem e executa o primeiro bloco onde a condição for verdadeira.

Sintaxe: 

if (condição1) {
    # Código se a condição1 for verdadeira
}
elseif (condição2) {
    # Código se a condição2 for verdadeira
}
else {
    # Código se nenhuma condição anterior for verdadeira
}

Exemplo: 

$nota = 15

if ($nota -ge 18) {
    Write-Output "Excelente!"
}
elseif ($nota -ge 14) {
    Write-Output "Bom!"
}
elseif ($nota -ge 10) {
    Write-Output "Suficiente."
}
else {
    Write-Output "Reprovado."
}

5.3. Dicas práticas

  • Usa parênteses () à volta da condição.
  • Usa chavetas {} sempre, mesmo para blocos de uma só linha.
  • Mantém o código indentado para melhor leitura.
  • Começa sempre pelos casos mais prováveis ou mais importantes no if.
  • Se houver apenas dois caminhos possíveis, usa if/else.
  • Se houver vários caminhos possíveis, usa if/elseif/else.

5.4. 🧩 Exercícios práticos – if, else, elseif

🧩 Exercício 1 – Maior ou menor de idade

  1. Pede ao utilizador a idade (Read-Host).
  2. Se a idade for maior ou igual a 18, mostra “És maior de idade.”
  3. Caso contrário, mostra “És menor de idade.”
Resposta 
$idade = Read-Host "Introduz a tua idade"

if ($idade -ge 18) {
    Write-Output "És maior de idade."
}
else {
    Write-Output "És menor de idade."
}

🧩 Exercício 2 – Classificação de notas

  1. Pede ao utilizador uma nota de 0 a 20 (Read-Host).
  2. Mostra:
    • Se a nota for maior ou igual a 18 → “Excelente!”
    • Se for maior ou igual a 14 → “Bom!”
    • Se for maior ou igual a 10 → “Suficiente.”
    • Caso contrário → “Reprovado.”
Resposta 
$nota = Read-Host "Introduz a tua nota"

if ($nota -ge 18) {
    Write-Output "Excelente!"
}
elseif ($nota -ge 14) {
    Write-Output "Bom!"
}
elseif ($nota -ge 10) {
    Write-Output "Suficiente."
}
else {
    Write-Output "Reprovado."
}

🧩 Exercício 3 – Verificar se o ano é bissexto

  1. Pede ao utilizador um ano (Read-Host).
  2. Verifica se é bissexto:
    • Divisível por 4 e não por 100, ou divisível por 400.
  3. Mostra:
    • “Ano bissexto.” ou “Ano comum.”
Resposta 
$ano = Read-Host "Introduz um ano"

if ((($ano % 4) -eq 0 -and ($ano % 100) -ne 0) -or (($ano % 400) -eq 0)) {
    Write-Output "Ano bissexto."
}
else {
    Write-Output "Ano comum."
}

🧩 Exercício 4 – Contar processos ativos

  1. Usa Get-Process para listar os processos.
  2. Conta quantos processos estão ativos.
  3. Se houver mais de 100 processos:
    • Mostra “Muitos processos!”
  4. Caso contrário:
    • Mostra “Número normal de processos.”
Resposta 
$processos = Get-Process
$total = $processos.Count

if ($total -gt 100) {
    Write-Output "Muitos processos!"
}
else {
    Write-Output "Número normal de processos."
}

🧩 Exercício 5 – Validação de palavra-passe

  1. Pede ao utilizador uma palavra-passe (Read-Host).
  2. Se a palavra-passe tiver menos de 6 caracteres, mostra “Palavra-passe muito curta.”
  3. Se tiver entre 6 e 10 caracteres, mostra “Palavra-passe aceitável.”
  4. Se tiver mais de 10 caracteres, mostra “Palavra-passe forte.”
Resposta 
$password = Read-Host "Introduz a tua palavra-passe"

if ($password.Length -lt 6) {
    Write-Output "Palavra-passe muito curta."
}
elseif ($password.Length -le 10) {
    Write-Output "Palavra-passe aceitável."
}
else {
    Write-Output "Palavra-passe forte."
}

6. Resumo Final e Boas Práticas

6.1. O que aprendeste nesta sessão

  • Como usar estruturas condicionais (if, else, elseif) para tomar decisões em scripts.
  • Como trabalhar com operadores de comparação (-eq, -ne, -gt, -lt, etc.).
  • Como usar operadores lógicos (-and, -or).
  • Como receber inputs do utilizador (Read-Host).
  • Como validar texto usando expressões regulares (-match).

6.2. Boas práticas ao usar if, else, elseif

✅ Usa parênteses () nas condições:
Ajuda a ler melhor e evita erros.

✅ Usa chavetas {} sempre, mesmo para uma linha:
Dá clareza e evita confusões em scripts maiores.

✅ Indenta o código dentro dos blocos:
Facilita a leitura, manutenção e debugging.

✅ Começa com o caso mais provável:
Assim o script executa mais rápido em cenários comuns.

✅ Usa else apenas se fizer sentido:
Se não houver uma ação clara para o caso contrário, o else pode ser evitado.

✅ Testa sempre todos os caminhos:
Garante que o teu script funciona corretamente em todos os cenários.

7. Estruturas switch

Aprendeste a usar if, else e elseif para tomar decisões no teu script.
Agora vais aprender uma outra estrutura condicional: o switch.

  • Permite comparar uma variável com vários valores diferentes.
  • Permite usar wildcards (*) e expressões regulares (padrões mais avançados).

7.1. Sintaxe básica do switch 

switch ($variavel) {
    "valor1" { # Código a executar se $variavel for "valor1" }
    "valor2" { # Código a executar se $variavel for "valor2" }
    default  { # Código a executar se não corresponder a nenhum valor }
}
  • O bloco default é opcional.

Exemplo básico:

$dia = "Terça"

switch ($dia) {
    "Segunda" { Write-Output "Início da semana." }
    "Terça"   { Write-Output "Segundo dia de trabalho." }
    "Quarta"  { Write-Output "Metade da semana!" }
    default   { Write-Output "Outro dia." }
}

7.2. switch -Wildcard

Se quiseres usar wildcards (*, ?, etc.), basta acrescentares o parâmetro -Wildcard.

switch -Wildcard ($variavel) {
    "A*" { # Começa por A }
    "*Z" { # Termina em Z }
    default { # Nenhum dos padrões }
}

Exemplo:

$palavra = "Ana"

switch -Wildcard ($palavra) {
    "A*" { Write-Output "Começa por A" }
    "*z" { Write-Output "Termina em z" }
    default { Write-Output "Outro padrão" }
}

7.3. switch -Regex

Se quiseres comparar usando padrões de expressões regulares, usa -Regex.

switch -Regex ($variavel) {
    "^\d+$" { # Só números }
    "^[A-Z]" { # Começa por maiúscula }
    default  { # Outro caso }
}

Exemplo:

$texto = "12345"

switch -Regex ($texto) {
    "^\d+$" { Write-Output "É um número!" }
    "^[A-Z]" { Write-Output "Começa com letra maiúscula!" }
    default { Write-Output "Outro padrão" }
}

Expressões lógicas dentro de um switch

Para além de comparar valores diretamente, o switch também pode usar expressões booleanas personalizadas

Cada bloco pode conter uma condição lógica que é avaliada como $true ou $false.
Quando a condição for verdadeira, o bloco correspondente é executado.

Exemplo: Categorizar fruta com base em arrays 

$dia = Read-Host "Introduz um dia da semana"

switch ($dia) {
    { $_ -in @("segunda", "terça", "quarta", "quinta", "sexta") } { Write-Output "Dia útil" }
    { $_ -in @("sábado", "domingo") }                              { Write-Output "Fim de semana" }
    default                                                        { Write-Output "Dia inválido" }
}

Neste exemplo:

  • $_ representa o valor inserido ($dia).
  • O operador -in verifica se esse valor pertence a um dos arrays definidos.
  • Cada condição devolve $true ou $false, e o primeiro bloco que for verdadeiro é executado. Este tipo de uso é útil quando queres agrupar condições complexas ou verificar se o valor pertence a uma coleção de possibilidades, sem escrever vários case separados.

7.4. 🧩 Exercícios práticos – switch

🧩 Exercício – Classificar frutas

  1. Cria uma variável $fruta.
  2. Usa switch para:
    • Mostrar “Fruta cítrica” se for “laranja”, “limão” ou “tangerina”.
    • Mostrar “Fruta tropical” se for “banana”, “manga” ou “ananás”.
    • Mostrar “Outra fruta” caso contrário.
Resposta 
$fruta = "manga"

switch ($fruta) {
    "laranja" { Write-Output "Fruta cítrica" }
    "limão" { Write-Output "Fruta cítrica" }
    "tangerina" { Write-Output "Fruta cítrica" }
    "banana" { Write-Output "Fruta tropical" }
    "manga" { Write-Output "Fruta tropical" }
    "ananás" { Write-Output "Fruta tropical" }
    default { Write-Output "Outra fruta" }
}

🧩 Exercício – Validar entradas usando padrões

  1. Pede ao utilizador para escrever uma palavra.
  2. Usa switch -Regex para:
    • Mostrar “Número encontrado” se for apenas números (^\d+$).
    • Mostrar “Começa com letra maiúscula” se começar por letra maiúscula (^[A-Z]).
    • Mostrar “Outro tipo de entrada” caso contrário.
Resposta 
$entrada = Read-Host "Introduz uma palavra"

switch -Regex ($entrada) {
    "^\d+$" { Write-Output "Número encontrado" }
    "^[A-Z]" { Write-Output "Começa com letra maiúscula" }
    default { Write-Output "Outro tipo de entrada" }
}

🧩 Exercício – Categorizar idade

  1. Pede ao utilizador para introduzir a idade.
  2. Usa switch com expressões lógicas para:
    • Mostrar “Menor de idade” se for menor que 18.
    • Mostrar “Adulto” se estiver entre 18 e 64.
    • Mostrar “Senior” se tiver 65 ou mais.
Resposta 
$idade = Read-Host "Introduz a tua idade"

switch ($idade) {
    { $_ -lt 18 } { Write-Output "Menor de idade" }
    { $_ -ge 18 -and $_ -lt 65 } { Write-Output "Adulto" }
    { $_ -ge 65 } { Write-Output "Senior" }
    default { Write-Output "Valor inválido" }
}

Resumo – Estruturas switch

O que aprendeste nesta sessão:

  • Como usar o comando switch para simplificar múltiplas condições.
  • Como usar switch -Wildcard para trabalhar com padrões de texto que incluem * e ?.
  • Como usar switch -Regex para testar expressões regulares de forma elegante.
  • Boas práticas no uso do switch para tornar o código mais limpo e fácil de manter.

Boas práticas ao usar switch:

✅ Usa switch quando tens várias comparações possíveis para o mesmo valor.

✅ Prefere switch -Wildcard para comparar terminações ou padrões simples (ex: “*.pdf”).

✅ Prefere switch -Regex quando precisas de correspondências mais poderosas (ex: validar se é um número).

✅ Inclui sempre um default para tratar casos não previstos.

✅ Mantém o código dentro de cada bloco {} simples e bem indentado.
