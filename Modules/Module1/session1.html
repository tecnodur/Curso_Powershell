🧭 Sessão 1: Conceitos Básicos e Configuração


🎯 Objetivo Introduzir os conceitos básicos de PowerShell e Shell Scripting, compará-los, e configurar o ambiente de trabalho.

1. Introdução (20 min)

1.1. O que é um Shell?

  • Um Shell é um programa que serve como interface entre o utilizador e o sistema operacional. Ele recebe comandos do utilizador, interpreta esses comandos e instrui o sistema operacional a executá-los. Os resultados da execução são então exibidos de volta ao utilizador.
  • Existem dois tipos principais de Shells:

Shells de Linha de Comando (CLI - Command Line Interface):

  • Interfaces textuais onde o usuário interage digitando comandos e recebendo respostas em forma de texto.
  • Exemplos: PowerShell, Bash, cmd.exe (Prompt de Comando do Windows).

Shells Gráficos (GUI - Graphical User Interface):

  • Interfaces visuais com ícones, janelas e menus.
  • Exemplos: Windows Explorer, Finder no macOS.

1.2. O que é PowerShell?

  • PowerShell é um exemplo de shell de linha de comando e uma linguagem de script integrados em um único ambiente.
  • Foi inicialmente projetado como uma framework para automatizar tarefas administrativas no Windows, mas evoluiu para uma ferramenta multiplataforma disponível em Windows, Linux e MacOS.
  • Atualmente, é utilizado para diversos tipos de tarefas, como administração de sistemas, automação de fluxos de trabalho e integração com serviços na cloud.
  • O PowerShell oferece muitos comandos úteis, mas suas capacidades podem ser expandidas a qualquer momento pela instalação de módulos adicionais.
  • Baseia-se em .NET e inclui uma linguagem de scripting, bem como um interpretador de linha de comandos.
  • Permite a automação de tarefas administrativas no Windows, Linux e MacOS.

1.3. Diferença entre CMD e PowerShell no Windows

O CMD (Command Prompt) e o PowerShell são ambos Shells de linha de comando no Windows, mas têm diferenças fundamentais em termos de capacidade, sintaxe, funcionalidade e uso.

Aspecto CMD (Command Prompt) PowerShell
Lançamento Década de 1980 (MS-DOS, Windows NT) 2006 (Windows PowerShell), 2016 (PowerShell Core)
Linguagem Não é uma linguagem de programação completa. Linguagem de scripting poderosa e orientada a objetos.
Comandos Baseados em MS-DOS e Windows NT. Ex.: dir, copy, del. Usa cmdlets (Get-Process, Get-Service) e aceita comandos do CMD.
Suporte a Objetos Trabalha apenas com texto puro. Trabalha com objetos .NET, permitindo manipulação avançada de dados.
Automação Limitada, mais adequada para tarefas básicas. Ideal para automação de sistemas, CI/CD, gerenciamento de servidores e cloud.
Pipeline Simples (envia texto de um comando para outro). Complexo e poderoso (envia objetos completos entre cmdlets).
Compatibilidade Compatível com comandos antigos do MS-DOS. Compatível com comandos do CMD, mas oferece muito mais funcionalidades.
Integração Focado apenas no sistema operacional local. Pode gerenciar sistemas locais, remotos e na nuvem (Azure, AWS).
Editor Integrado Não possui. Inclui PowerShell ISE e é totalmente integrado ao Visual Studio Code.
Exemplos de Uso dir, ipconfig, netstat. Get-Process, Get-Service, Get-Command, Invoke-RestMethod.

1.4. 🚀 Resumo

  • CMD: É antigo, limitado e trabalha apenas com texto puro. Ideal para tarefas básicas e rápidas.
  • PowerShell: É moderno, poderoso e trabalha com objetos, facilitando automação avançada, administração de sistemas e integração com plataformas na nuvem.

1.5. O que é Shell Scripting?

  • Shell Scripting é um método de automatizar tarefas por meio de scripts executados pelo Shell do sistema operacional.
  • Amplamente utilizado em sistemas Unix/Linux, mas também disponível no Windows (especialmente com o WSL - Windows Subsystem for Linux).
  • Inclui linguagens como Bash, Zsh, e outras.

1.6. Casos de uso típicos de cada linguagem:

  • PowerShell: Administração de sistemas Windows, automatização de tarefas de infraestrutura em Azure, manipulação de arquivos e relatórios.
  • Bash: Gestão de servidores Linux, manipulação de arquivos e diretórios, automatização de backups e processos de instalação.

1.7. Funcionalidades do PowerShell

  1. Sistema de Ajuda Integrado: 
    Get-Help Get-Process
  2. Pipeline Poderoso: 
    Get-Process | Where-Object {$_.CPU -gt 100}
  3. Aliases:
    Exemplo: ls, cls, cd funcionam no PowerShell.

1.8. Como o PowerShell é Diferente de Shells Tradicionais

  • Trabalha com objetos, não com texto.
  • Usa cmdlets com sintaxe Verbo-Substantivo.
  • Diferentes tipos de comandos: executáveis, cmdlets, funções, aliases.

2. Instalação e Configuração do Ambiente

2.1. Windows PowerShell vs. PowerShell Core

Windows PowerShell

  • Plataforma: Apenas Windows.
  • Baseado em: .NET Framework (antigo e obsoleto para novos projetos).
  • Última versão: 5.1 (Lançada em 2016, com últimas atualizações de segurança em 2019).
  • Instalação: Pré-instalado em versões modernas do Windows (Windows 7 SP1, Windows Server 2008 R2 e posteriores).
  • Objetivo Principal: Administração de sistemas Windows e automação de tarefas locais.
  • Limitações:
    • Apenas disponível no Windows, sem suporte para Linux ou MacOS.
    • Não suporta .NET Core ou versões modernas de .NET.
    • Desenvolvimento interrompido, apenas com atualizações de segurança.
  • Recomendação: Usar apenas para manter compatibilidade com sistemas antigos ou quando funcionalidades específicas do Windows PowerShell são necessárias.

PowerShell Core (Agora chamado apenas de PowerShell)

  • Plataforma: Multiplataforma (Windows, Linux, MacOS).
  • Baseado em: .NET Core / .NET 5+ (mais recente, mais eficiente e de código aberto).
  • Última versão: 7.x (Atualizado regularmente).
  • Instalação: Disponível para download no GitHub.
  • Objetivo Principal: Automação e administração de sistemas em múltiplas plataformas, incluindo integração com serviços em cloud (Azure, AWS, etc).
  • Vantagens:
    • Funciona em múltiplos sistemas operacionais.
    • Melhor desempenho, menor consumo de memória e suporte para APIs modernas do .NET.
    • Desenvolvimento ativo e contínuo com novas funcionalidades.
    • Melhor compatibilidade com plataformas cloud e ferramentas DevOps.
  • Recomendação: Ideal para novos projetos, especialmente quando a compatibilidade multiplataforma é necessária.

💡 Nota: Desde a versão 7.0, o PowerShell Core é simplesmente chamado de PowerShell.

2.2. Bash no Linux, MacOS e Windows (WSL)

Linux e MacOS

  • Disponibilidade: O Bash é instalado por padrão em praticamente todas as distribuições Linux e no MacOS.
  • Alternativas: Pode ser substituído por shells modernos como zsh, fish, ou ksh.
  • Popularidade: Especialmente comum em sistemas Linux, onde é usado para tarefas administrativas, scripts de inicialização e automação.
  • Atualização: No Linux, pode ser atualizado usando o sistema de pacotes da distribuição (ex.: apt, yum, dnf).

Windows (WSL - Windows Subsystem for Linux)

  • Disponibilidade: Disponível nativamente no Windows 10 e 11.
  • Objetivo: Permitir que os utilizadores executem um ambiente Linux diretamente no Windows sem necessidade de uma máquina virtual completa.
  • Vantagens:
    • Permite desenvolver e testar scripts Bash nativamente no Windows.
    • Fácil integração com ferramentas de desenvolvimento modernas (ex.: Docker).
    • Melhor suporte para automação e ferramentas Linux em ambientes Windows.

3. Como Instalar e Configurar PowerShell Core e Bash

3.1. Instalar PowerShell (Core)

  • Windows (WinGet - Recomendado): 
    winget install --id Microsoft.PowerShell --source winget
  • Windows (MSI Installer):
    • Baixar o instalador mais recente do GitHub Releases.
    • Executar o instalador e seguir as instruções.
  • Linux (Debian/Ubuntu): 
    sudo apt-get install -y powershell
  • Linux (Fedora/CentOS/RHEL): 
    sudo dnf install -y powershell
  • MacOS (via Homebrew): 
    brew install --cask powershell
  • Executar o PowerShell: 
    pwsh

3.2. Instalar Bash no Windows (WSL)

  • Habilitar o WSL: 
    wsl --install
  • Listar distribuições disponiveis 
      wsl --list --online
  • Instalar uma distribuição específica (ex.: Ubuntu): 
    wsl --install -d Ubuntu
  • Verificar a versão do WSL instalada: 
    wsl -l -v
  • Atualizar o WSL para a versão 2: 
    wsl --set-default-version 2
  • Executar o Bash no Windows: 
    wsl

    3.3. Comandos úteis do WSL

  • wsl -l: Lista todas as distribuições WSL instaladas.

  • wsl -d <distribution name>: Inicia uma nova sessão WSL usando a distribuição especificada.

  • wsl --shutdown: Encerra todas as sessões WSL em execução.

  • wsl --terminate <distribution name>: Termina a sessão WSL da distribuição especificada.

  • wsl --default <distribution name> ou wsl --setdefault <distribution name>: Define a distribuição WSL padrão (usada quando se executa apenas wsl no terminal).

  • wsl --terminate <distribution name>: Termina uma distribuição WSL em execução.

  • wsl --shutdown: Encerra todas as instâncias WSL em execução.
