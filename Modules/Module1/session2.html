1. 📖 Ajuda e Descoberta de Comandos

Uma das grandes forças do PowerShell é a sua documentação integrada. Comandos como Get-Help e Get-Command permitem ao utilizador aprender diretamente a partir da consola.

ℹ️ Nota Importante:
Se vires uma mensagem como: “Cannot find the Help files for this cmdlet on this computer. It is displaying only partial help.” Significa que o PowerShell está a mostrar apenas uma versão resumida. Podes:

  • Tentar usar Update-Help para descarregar a versão completa (se tiveres permissões)
  • Usar Update-Help -UICulture en-US -ErrorAction SilentlyContinue para contornar restrições de idioma ou permissões
  • Usar o modo online: Get-Help Get-Process -Online

1.1. 🧪 Exemplos 

Get-Help Get-Process               # Mostra a ajuda básica
Get-Help Get-Process -Examples     # Mostra exemplos práticos
Get-Help Get-Process -Online       # Abre no browser
Get-Command *service*              # Procura comandos relacionados
Get-Help Get-Service -Detailed     # Mostra informação detalhada

1.2. 🔍 Como ler o output do Help 

cls
Get-Help Get-Service -Detailed

Este comando permite verificar:

  • Name: nome do cmdlet
  • Synopsis: descrição breve
  • Syntax: formas de utilização
  • Description: descrição detalhada
  • Parameters: explicação de cada parâmetro
  • Remarks: observações adicionais, como compatibilidade
cls
Get-Help Get-Service -Examples

Este comando premite verificar os exemplos de utilização práticos.

1.3. 🔎 Comando para inspecionar objetos 

Get-Service | Get-Member

O Get-Member permite ver as propriedades (ex: Status, DisplayName) e métodos dos objetos devolvidos.

Get-Service | Select-Object Status, DisplayName, ServiceType

1.4. 🧩 Exercícios

Neste exercício vais aprender a usar a ajuda integrada no PowerShell para perceber como um comando funciona.

Passos:

  1. Abre o PowerShell
  2. Corre o seguinte comando para ver uma lista de todos os serviços no sistema: 
      Get-Service

    Depois, experimenta filtrar por nome:

    • Um serviço específico: 
      Get-Service wuauserv
    • Usando um wildcard (por exemplo, todos os que começam por “c”): 
       Get-Service c*
  3. Escreve o seguinte comando para veres a ajuda básica: Get-Help Get-Service
  4. Vê os exemplos com: Get-Help Get-Service -Examples
  5. Tenta abrir a ajuda online (se possível): Get-Help Get-Service -Online
  6. Usa o Get-Member para descobrir que propriedades existem em objetos de serviço. Depois usa Select-Object para escolher apenas as propriedades que permitam ver o Status, Nome do utilizador e tipo de serviço.
    Exemplo de solução 
    Get-Service | Get-Member
Get-Service | Select-Object Status, UserName, ServiceType
  7. Cria um comando que obtenha os detalhes de um ou mais processos que costumas usar, como chrome, notepad, ou outro processo que esteja a correr no teu sistema. Queremos listar os seguintes detalhes:
    • Nome do processo
    • ID (PID - {completa})
    • Utilização de CPU (CPU - mostra o tempo total de CPU utilizado (em segundos).)
    • Memória utilizada (Working Set - mostra a memória física (em bytes) usada pelo processo.)
    • Tempo de execução (StartTime - {completa})
    • Estado (Responding - indica se a aplicação está a responder (True ou False))
    Exemplo de solução 
    Get-Process chrome | Select-Object Name, Id, CPU, WorkingSet, StartTime, Responding

💡 Dica: Experimenta com diferentes nomes de processos (ex: notepad, powershell, explorer) e compara os valores.

2. 🔌 Providers

No PowerShell, providers são interfaces que permitem aceder, navegar e manipular diferentes tipos de armazenamento (como o sistema de ficheiros, o registo do Windows ou variáveis de ambiente) usando comandos comuns como Get-ChildItem ou Set-Location, tal como farias com pastas e ficheiros.

Ou seja, mesmo que estejas a trabalhar com algo que não é um disco (como variáveis ou chaves de registo), podes usar os mesmos comandos de navegação como se estivesses a explorar um sistema de pastas. Podes ver todos os providers disponíveis no teu sistema com o seguinte comando:

   Get-PSProvider

2.1. 📂 FileSystem

Permite interagir com pastas e ficheiros no disco, tal como estamos habituados.
Este é o provider usado por omissão quando abrimos o PowerShell.

2.2. 🧬 Registry

Permite navegar no registo do Windows como se fosse uma estrutura de pastas. Tem dois “drives”: HKLM: (Local Machine) e HKCU: (Current User). Exemplos:

Set-Location HKCU:\Software
Get-ChildItem HKLM:\SYSTEM

2.3. 🌱 Environment

Permite aceder e modificar variáveis de ambiente (como PATH, TEMP, USERNAME, etc). Exemplos:

Get-ChildItem Env:
$Env:USERNAME
$Env:TEMP

2.4. 🏷️ Alias

Permite listar e criar atalhos (aliases) para comandos do PowerShell. Exemplos:

Get-ChildItem Alias:
Get-Alias ls
New-Alias listar Get-ChildItem

2.5. ✅ Outros providers

Consoante o sistema ou módulos carregados, podem existir outros providers disponíveis. Por exemplo:

  • Function: – acede às funções definidas no PowerShell.
  • Variable: – acede às variáveis em memória.

Para mais detalhes sobre providers no PowerShell, consulta a documentação oficial:
🔗 about_Providers - PowerShell Documentation

3. 📁 Navegação no Sistema de Ficheiros

3.1. 🧪 Comandos úteis 

Get-Location     # Mostra a localização atual (pwd)
Set-Location ..  # Sobe um nível (cd ..)
Get-ChildItem    # Lista ficheiros e pastas (ls)
Clear-Host       # Limpa o terminal (cls)
Push-Location    # Guarda a localização atual
Pop-Location     # Volta à localização guardada

3.2. 🌐 Dicas

  • -Force mostra ficheiros ocultos.
  • Caminhos relativos (ex: ..\Documentos) mudam com a localização atual.
  • Caminhos absolutos (ex: C:\Users\Nome\Documentos) são sempre os mesmos.
  • Usa ~ para ir para a tua pasta pessoal.

3.3. 🧩 Exercícios

Nota: Usa sempre os cmdlets do PowerShell como Set-Location ou Get-ChildItem em vez dos comandos do bash como cd ou ls.

Neste exercício vais praticar os comandos de navegação no sistema de ficheiros no PowerShell.

Passos:

  1. Abre o PowerShell.

  2. Vai para a tua pasta pessoal:
    Código 
    Set-Location ~

  3. Corre o seguinte comando para criar a estrutura de pastas:

    New-Item -Path "~\CursoPS\Modulo1\Exemplos" -ItemType Directory -Force
  4. Verifica a estrutura final de pastas executando o seguinte: 
    Set-Location ~
Get-ChildItem -Recurse "CursoPS"

    Estrutura esperada:

    
CursoPS
│
└── Modulo1
   │
   └── Exemplos
  5. Navega até à pasta CursoPS:
    Código 
    Set-Location "~\CursoPS"
  6. Navega até à pasta Modulo1 usando caminho relativo:
    Código 
    Set-Location .\Modulo1
  7. Agora navega até à pasta Exemplos usando:
    • Caminho relativo:
      Código 
       Set-Location .\Exemplos
    • Caminho absoluto:
      Código 
       Set-Location "~\CursoPS\Modulo1\Exemplos"
  8. Testa a navegação com Push-Location e Pop-Location:
    • Guarda a localização atual, vai para outro local, e volta: 
      Push-Location
Set-Location ~
Pop-Location

4. 🌱 Variáveis de ambiente

O provider Environment permite aceder e manipular variáveis de ambiente do sistema operativo diretamente no PowerShell. Estas variáveis são usadas por aplicações e scripts para obter informações como o nome do utilizador, a pasta temporária, o caminho de instalação, entre outras.
Os restantes tipos de variáveis são vistos na sessão 3

4.1. 📋 Consultar variáveis

Podes ver todas as variáveis de ambiente com:

Get-ChildItem Env:

Para consultar uma variável específica:

   $Env:USERNAME
   $Env:PATH
   $Env:TEMP

4.2. ✏️ Alterar ou definir variáveis (temporariamente)

Podes criar ou alterar variáveis diretamente:

   $Env:NOVAVARIAVEL = "Valor de teste"
   $Env:PATH += ";C:\Ferramentas"

⚠️ Estas alterações só duram enquanto a sessão do PowerShell estiver ativa.

4.3. 🧼 Remover variáveis

Podes remover uma variável da sessão com:

   Remove-Item Env:NOVAVARIAVEL

4.4. 🧩 Exercícios

Vamos experimentar criar e aceder a variáveis de ambiente.

  1. Cria uma nova variável chamada “CURSO” e com o valor “PowerShell & Shell Scripting”
    Código 
       $Env:CURSO = "PowerShell & Shell Scripting"
  2. Verifica o valor:
    Código 
       $Env:CURSO
  3. Acede à variável usando letras minúsculas: 
    $Env:curso

    O que observas?

    Explicação

    No PowerShell, as variáveis de ambiente não são sensíveis a maiúsculas ou minúsculas.
    Ou seja, $Env:CURSO, $Env:curso e $Env:Curso referem-se exatamente à mesma variável.

    Isto acontece porque o provider Environment está ligado ao sistema operativo Windows, que também não distingue entre maiúsculas e minúsculas neste contexto.

  4. Remove a variável: 
       Remove-Item Env:CURSO

  5. Refaz os exercícios do ponto 3 (navegar no sistema de ficheiros), mas em vez de usares ~ para aceder à tua pasta pessoal, usa a variável de ambiente $Env:USERPROFILE.

    • Vai para a tua pasta pessoal: 
      Set-Location $Env:USERPROFILE
    • Se necessário cria novamente a estrutura de pastas: 
      New-Item -Path "$Env:USERPROFILE\CursoPS\Modulo1\Exemplos" -ItemType Directory -Force
    • Explora a estrutura usando comandos de navegação como Set-Location, Get-Location e Push-Location.

💡 Dica: Podes confirmar que $Env:USERPROFILE e ~ apontam para o mesmo local com:

   "$Env:USERPROFILE"
   Resolve-Path ~

ℹ️ Nota Importante: As variáveis definidas ou alteradas com $Env: só afetam a sessão atual do PowerShell.
Se quiseres definir variáveis de ambiente de forma permanente (por exemplo, para todas as sessões), precisas de usar ferramentas do sistema operativo ou editar as definições no Painel de Controlo (Windows) ou via registo (Registry).

Podes combinar este provider com outros comandos para criar scripts mais dinâmicos e adaptáveis ao ambiente onde estão a correr.

4.5. 🔍 Diferença entre $ e $Env:

Quando usas apenas o símbolo $, estás a trabalhar com variáveis normais do PowerShell, que existem apenas na sessão atual e são controladas pelo próprio PowerShell.

Quando usas $Env:, estás a aceder a variáveis de ambiente do sistema operativo, através do provider Environment.

Comparação:

Exemplo Tipo de variável Persistência Acessível fora do PowerShell?
$nome = "Joana" Variável do PowerShell Sessão atual ❌ Não
$Env:NOME = "Joana" Variável de ambiente Sessão atual (ou permanente, se configurada no SO) ✅ Sim (herdada por apps lançadas a partir desta sessão)

🔍 Verificação prática: Para comprovar a diferença entre variáveis normais e variáveis de ambiente fora do PowerShell: Cria as duas variáveis no PowerShell:

$normal = "sou normal"  
$Env:AMBIENTE = "sou ambiente"

Abre uma nova sessão do CMD a partir do PowerShell:

cmd /d /k echo %normal% 

cmd /d /k echo %AMBIENTE%

  • /d – Desativa a execução automática de comandos definidos na variável de ambiente AutoRun.
    Útil para garantir que o CMD começa com um ambiente “limpo”, sem comandos automáticos.

  • /k – Executa o comando indicado e mantém a janela do CMD aberta depois.
    Se usasses /c em vez de /k, o CMD fechar-se-ia logo após o echo.

Observa o resultado:

%normal%
sou ambiente

✅ Explicação:

%normal% aparece como texto literal porque $normal é uma variável interna do PowerShell — não é herdada por outras aplicações ou shells.

%AMBIENTE% mostra o valor correto porque $Env:AMBIENTE é uma variável de ambiente, e essas são automaticamente visíveis por processos filhos como o CMD.

5. 🏷️ Alias

No PowerShell, um alias é um nome alternativo (geralmente mais curto) para um comando.

Isto permite escrever comandos de forma mais rápida ou usar nomes familiares (por exemplo, do Bash).
Exemplo: ls é um alias para Get-ChildItem.

Podes ver todos os aliases disponíveis com:

   Get-Alias

5.1. 🔎 Exemplos simples

Verifica o comando por trás de um alias:

Get-Alias ls

Podes ver todos os aliases que apontam para Get-ChildItem com:

Get-Alias | Where-Object {$_.Definition -eq 'Get-ChildItem'}

Resultado será algo como:

  • dir
  • gci
  • ls

Cria um alias personalizado:

New-Alias listar Get-ChildItem

Usa o alias que criaste:

listar

Remove o alias:

Remove-Item Alias:listar

⚠️ Aliases personalizados só duram durante a sessão atual. Mais à frente veremos como personalizar o profile do utilizador e criar aliases persistentes.

🔗 Documentação oficial
about_Aliases - PowerShell Documentation

5.2. 🧩 Exercícios

  1. Descobre que comando está por trás dos seguintes aliases:
    • ls
    • pwd
    • cls
  2. Cria um alias chamado meuservicos para o comando Get-Service.
Resposta 
New-Alias meuservicos Get-Service
  1. Corre o alias meuservicos e confirma que funciona. 
    meuservicos
  2. Remove o alias meuservicos.
Resposta 
Remove-Item Alias:meuservicos
  1. Cria um alias chamado procativos para o comando Get-Process.
Resposta

procativos

💡 Dica: Como os aliases não aceitam argumentos ou pipelines, devem ser usados apenas para comandos simples. Para comandos mais complexos, como Get-Service | Select-Object …, considera criar uma função.
